  คู่มือตั้งค่า Firebase สำหรับระบบจองแบดมินตัน
═══════════════════════════════════════════════════════════════

ใช้เวลาประมาณ 5-10 นาที ฟรีไม่มีค่าใช้จ่าย


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ขั้นที่ 1: สร้างโปรเจกต์ Firebase
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. เปิด  https://console.firebase.google.com
2. ล็อกอินด้วย Google account
3. กด "Create a project" (สร้างโปรเจกต์)
4. ตั้งชื่อโปรเจกต์ เช่น "badminton-booking"
5. ขั้นถัดไปจะถามเรื่อง Google Analytics — เลือก "Disable" ได้ ไม่จำเป็น
6. กด "Create project" รอสักครู่จนเสร็จ


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ขั้นที่ 2: เปิดใช้งาน Firestore Database
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. ในเมนูซ้ายมือ กด "Build" → "Firestore Database"
2. กดปุ่ม "Create database"
3. เลือก location: "asia-southeast1 (Singapore)" หรือใกล้สุด
4. กด "Next"
5. เลือก "Start in test mode" (สำคัญ!)
   ※ test mode จะให้ทุกคนอ่าน-เขียนได้ 30 วัน เหมาะกับการใช้กลุ่มเพื่อน
6. กด "Create"


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ขั้นที่ 3: ตั้ง Security Rules (สำคัญมาก!)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

หลังจากสร้าง Firestore แล้ว:

1. กดแท็บ "Rules" ในหน้า Firestore
2. ลบโค้ดเก่าออก แล้ววางโค้ดนี้แทน:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /app/data {
      allow read, write: if true;
    }
  }
}

3. กด "Publish"

※ Rules นี้อนุญาตให้ทุกคนอ่าน-เขียน "เฉพาะที่ app/data" เท่านั้น
  คนอื่นจะลบ database ทั้งหมดของคุณไม่ได้


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ขั้นที่ 4: เพิ่ม Web App และคัดลอก Config
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. กดไอคอน ⚙ (Settings) ข้างคำว่า "Project Overview"
2. เลือก "Project settings"
3. เลื่อนลงไปหา "Your apps" → กดไอคอน </> (Web)
4. ตั้งชื่อ app เช่น "badminton-web"
5. ※ ไม่ต้องติ๊ก "Firebase Hosting"
6. กด "Register app"
7. จะเห็นโค้ด config หน้าตาแบบนี้:

// For Firebase JS SDK v7.20.0 and later, measurementId is optional
const firebaseConfig = {
  apiKey: "AIzaSyDYrliwQj9jmsbbS63ffpdIGPTDhJEGTAI",
  authDomain: "badminton-e2f4d.firebaseapp.com",
  projectId: "badminton-e2f4d",
  storageBucket: "badminton-e2f4d.firebasestorage.app",
  messagingSenderId: "289792798899",
  appId: "1:289792798899:web:faf8417a0eede18f2640c6",
  measurementId: "G-Q3EFXKVWKD"
};

   ※ คัดลอกเก็บไว้!


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ขั้นที่ 5: ใส่ Config เข้าไฟล์ HTML
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. เปิดไฟล์ "badminton-firebase.html" ด้วย text editor
   (Notepad, VS Code, TextEdit ก็ได้)

2. มองหาบรรทัด:

   // ↓↓↓ แทนที่ค่าด้านล่างด้วย config จาก Firebase Console ของคุณ ↓↓↓
   const firebaseConfig = {
     apiKey: "YOUR_API_KEY",
     ...
   };

3. แทนค่าทั้ง object firebaseConfig ด้วยที่คัดลอกมาจาก Firebase

4. เซฟไฟล์

5. เปิดด้วยเบราว์เซอร์ — ควรเชื่อมต่อสำเร็จ!


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ขั้นที่ 6: ทำให้เพื่อนเข้าใช้ได้
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

มี 3 ทางเลือก:

▸ ทางง่ายที่สุด: ส่งไฟล์ HTML ให้เพื่อนทุกคน
  → ใครเปิดก็เข้าระบบเดียวกัน (เพราะ config ชี้ไป Firebase อันเดียวกัน)

▸ ทางเหมาะกับมือถือ: อัปโหลดขึ้นโฮสต์ฟรี แล้วส่งลิงก์
  เช่น:
  • Netlify Drop  → https://app.netlify.com/drop
  • Vercel        → https://vercel.com
  • GitHub Pages  → https://pages.github.com
  • Cloudflare Pages → https://pages.cloudflare.com
  
  เพียงลากไฟล์ไปวางก็ได้ลิงก์ทันที ส่ง LINE ให้เพื่อนได้เลย

▸ ทางอื่น: เก็บใน Google Drive แบบ public link
  (ใช้ได้แต่ค่อนข้างยุ่งกว่า)


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ข้อมูลสำคัญที่ควรรู้
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

• รหัส Admin เริ่มต้น: badmin2025
  → เปลี่ยนได้ที่บรรทัด const ADMIN_PASSWORD = "badmin2025"; ในไฟล์ HTML

• Firebase free tier (Spark plan) ให้:
  - อ่าน 50,000 ครั้ง/วัน
  - เขียน 20,000 ครั้ง/วัน
  - เก็บข้อมูล 1 GB
  → เกินพอสำหรับกลุ่มเพื่อนใช้แบดมินตัน 100+ คน

• Test mode rules จะหมดอายุใน 30 วัน
  → แต่เราเปลี่ยนเป็น rules ของเราเองไปแล้วในขั้นที่ 3
  → ไม่ต้องกังวลเรื่องนี้

• ถ้าอยากเพิ่มความปลอดภัย (ขั้นสูง):
  → ใช้ Firebase Authentication + แก้ rules ให้อนุญาตเฉพาะ user ที่ login
  → ติดต่อเพื่อขอความช่วยเหลือเพิ่มเติม

  ปัญหาที่อาจเจอ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✗ เปิดไฟล์แล้วขึ้น "ต้องตั้งค่า Firebase ก่อนใช้งาน"
  → ยังไม่ได้แก้ firebaseConfig หรือยังคงค่า YOUR_API_KEY อยู่

✗ ขึ้น "ออฟไลน์" / "เชื่อมต่อ Firebase ไม่ได้"
  → ตรวจ Firestore rules ว่าตั้งตามขั้นที่ 3 หรือยัง
  → ลองรีเฟรชหน้า

✗ Console ของ browser (F12) ขึ้น error "Missing or insufficient permissions"
  → Firestore rules ผิด — กลับไปขั้นที่ 3 อ่านใหม่

✗ ลงชื่อแล้วเพื่อนไม่เห็น
  → ตรวจว่าทุกคนใช้ไฟล์ที่มี firebaseConfig เดียวกัน
  → ลองรีเฟรชหน้าฝั่งเพื่อน (โดยปกติจะ sync อัตโนมัติ)


เสร็จแล้ว! สนุกกับการตีแบดได้เลย 🏸
